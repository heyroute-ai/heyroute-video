# heyroute-video 构建方案

## 项目边界

这是一个面向公开分享的批量视频生产工具，不是 HeyNews 的内容仓库，也不是任何单一平台的自动发布机器人。

### 做什么

- 读取结构化 manifest，管理素材、场景、旁白和输出
- 将图片、PPT 页面、文字卡片和音频组合成时间线
- 通过统一接口接入本地或云端 TTS
- 生成/导入字幕并完成音画、尺寸、时长检查
- 输出视频、封面、字幕、元数据和可追踪的构建报告
- 为批量任务提供可恢复、可重跑的状态记录

### 暂不做什么

- 不内置任何账号密码、Cookie 或私有业务配置
- 不把某个平台的上传流程作为核心依赖
- 不把具体内容品牌、栏目文风或新闻采集放进核心库
- 不承诺自动绕过平台审核、版权或地区限制

## 分阶段路线

### Phase 0：规范和最小骨架

- 定义 `manifest v1`、场景模型和输出目录
- 建立 CLI、日志和错误码约定
- 加入一个无外部服务依赖的示例
- 用 FFmpeg 完成一条静态图片 + 音频 + 字幕的 9:16 渲染

### Phase 1：可用的单条生产链

- 支持图片、PPT 导出页和纯色文字场景
- 支持导入已有音频，以及一个 TTS provider
- 生成 SRT/VTT，支持按音频时长切分
- 增加渲染前后检查和 JSON 构建报告
- 提供 Windows、macOS、Linux 的安装说明

### Phase 2：批量与模板

- 批量 manifest、并发策略、失败重试和断点续跑
- 竖屏模板：新闻卡片、讲解卡片、图文时间线
- 字体、颜色、边距、片尾等主题配置
- 输出预览图和 contact sheet，便于人工验收

### Phase 3：可选发布适配器

- 先提供“生成发布包”的平台无关接口
- 再按贡献者维护独立的 Bilibili、YouTube 等适配器
- 发布器与核心渲染解耦，平台变更不影响本地生产

### Phase 4：开发者体验

- 本地 Web dashboard（仅在确有需求后加入）
- 插件机制和第三方模板注册
- 可选的远程任务队列，不把云服务作为使用前提

## 设计原则

1. **一份输入真相**：标题、场景、旁白、来源和输出路径来自 manifest，不在脚本参数里重复维护。
2. **可重现**：记录工具版本、渲染参数、素材哈希和 TTS 配置摘要。
3. **失败要显式**：缺素材、缺字体、音频时长异常、字幕越界都要报错，不静默猜测。
4. **核心无账号**：核心库不需要平台账号；凭据只由用户自己的发布适配器管理。
5. **先 CLI 后界面**：先把稳定的命令和数据结构做好，再决定是否加 dashboard。
6. **平台是插件**：平台上传逻辑不能反向污染渲染内核。

## 需要用户选择的决策

以下选项会影响 Phase 0 的实现顺序：

### 1. 核心技术栈

- **Python（推荐）**：最容易复用现有 `ahe-ai-video-workflow`、Whisper、Pillow 和 FFmpeg 经验。
- TypeScript/Bun：CLI 体验好，但媒体和语音生态需要重新评估。
- 混合：Python 负责媒体内核，TypeScript 做 CLI/界面，能力强但维护成本最高。

### 2. 第一条必须跑通的链路

- **HeyNews 竖屏新闻视频（推荐）**：直接验证当前真实需求，9:16、PPT 页面、配音、字幕、PNG/MP4。
- 通用图文讲解视频：更通用，但首个验收样例离现有业务更远。
- 两者同时做：早期抽象更完整，但第一版交付会变慢。

### 3. 输入格式

- **JSON/YAML manifest（推荐）**：适合程序批量生成和版本管理。
- Markdown + front matter：更适合人工编辑，但复杂时间线表达较弱。
- 两者都支持：体验最好，但要维护双向解析和一致性测试。

### 4. TTS 范围

- **先支持导入音频 + provider 接口（推荐）**：核心可离线测试，不锁定供应商。
- 首版直接接入一个本地 TTS：开箱即用，但安装和模型体积更重。
- 首版直接接入云 TTS：音质稳定，但需要 API key 和成本控制。

### 5. 发布边界

- **首版只生成发布包（推荐）**：视频、封面、字幕、标题、简介和 manifest，风险最低。
- 同时加入 Bilibili 发布器：更快形成完整闭环，但平台维护成本高。
- 从一开始做多平台：覆盖广，但会拖慢核心工具稳定性。

## 验收标准（Phase 1）

- 一条命令生成 1080×1920 MP4、SRT、封面和 JSON 报告
- 同一个 manifest 可重复执行，已有中间产物可复用
- 缺素材、缺音频、FFmpeg 失败时返回明确错误，不生成假成功结果
- 示例项目不包含真实账号、Cookie、私有 URL 或受限素材
- Windows CI 至少完成 manifest、时间线和校验层测试；FFmpeg 集成测试可按环境标记
