# heyroute-video 构建方案

## 项目边界

这是一个面向公开分享的批量视频生产工具，不是 HeyNews 的内容仓库，也不是任何单一平台的自动发布机器人。

### 做什么

- 读取结构化 manifest，管理素材、场景、旁白和输出
- 将图片、PPT 页面、文字卡片和音频组合成时间线
- 通过统一接口接入本地 TTS，云端 provider 不在首版范围内
- 生成/导入字幕并完成音画、尺寸、时长检查
- 输出视频、封面、字幕、元数据和可追踪的构建报告
- 为批量任务提供可恢复、可重跑的状态记录

### 暂不做什么

- 不内置任何账号密码、Cookie 或私有业务配置
- 不把某个平台的上传流程作为核心依赖
- 不把具体内容品牌、栏目文风或新闻采集放进核心库
- 不承诺自动绕过平台审核、版权或地区限制

## 分阶段路线

### Phase 0：规范和最小骨架

- 定义 `manifest v1`、场景模型和输出目录
- 建立 CLI、日志和错误码约定
- 加入一个无外部服务依赖的示例
- 用 FFmpeg 完成一条静态图片 + 音频 + 字幕的 9:16 渲染

### Phase 1：可用的单条生产链

- 支持图片、PPT 导出页和纯色文字场景
- 支持导入已有音频，以及一个 TTS provider
- 生成 SRT/VTT，支持按音频时长切分
- 增加渲染前后检查和 JSON 构建报告
- 提供 Windows、macOS、Linux 的安装说明

### Phase 2：批量与模板

- 批量 manifest、并发策略、失败重试和断点续跑
- 竖屏模板：新闻卡片、讲解卡片、图文时间线
- 字体、颜色、边距、片尾等主题配置
- 输出预览图和 contact sheet，便于人工验收

### Phase 3：可选发布适配器

- 先提供“生成发布包”的平台无关接口
- 再按贡献者维护独立的 Bilibili、YouTube 等适配器
- 发布器与核心渲染解耦，平台变更不影响本地生产

### Phase 4：开发者体验

- 本地 Web dashboard（仅在确有需求后加入）
- 插件机制和第三方模板注册
- 可选的远程任务队列，不把云服务作为使用前提

## 设计原则

1. **一份输入真相**：标题、场景、旁白、来源和输出路径来自 manifest，不在脚本参数里重复维护。
2. **可重现**：记录工具版本、渲染参数、素材哈希和 TTS 配置摘要。
3. **失败要显式**：缺素材、缺字体、音频时长异常、字幕越界都要报错，不静默猜测。
4. **核心无账号**：核心库不需要平台账号；凭据只由用户自己的发布适配器管理。
5. **先 CLI 后界面**：先把稳定的命令和数据结构做好，再决定是否加 dashboard。
6. **平台是插件**：平台上传逻辑不能反向污染渲染内核。

## 已确定决策

- **技术栈**：Python 3.12，FFmpeg 负责媒体处理，CLI 面向 Agent。
- **目标链路**：HeyNews 竖屏新闻视频，但核心 manifest 不绑定 HeyNews 的临时 JSON。
- **输入**：JSON/YAML manifest，支持单任务和 `jobs` 批量任务。
- **TTS**：本地 IndexTTS；适配、发现、健康检查、job 转换和缓存由本仓库负责，不接入云 TTS。
- **IndexTTS 边界**：不复制源码、模型或 `.venv`；通过 `E:\index-tts`、`HEYROUTE_INDEXTTS_HOME` 或 manifest 配置接入。
- **Agent 接口**：`doctor`、`validate`、`build`、`tts doctor`、`tts bootstrap`，支持 JSON 结果和 JSONL 事件。
- **发布范围**：只生成 MP4、封面、音频、SRT/VTT、标题、简介、标签和构建报告，不自动上传。
- **首版不做**：dashboard、MCP server、云 TTS、多平台发布器、远程任务队列和 HeyNews 专用强耦合转换。

详细字段定义见 [`docs/manifest-v1.md`](docs/manifest-v1.md)，IndexTTS 许可和运行边界见 [`docs/indextts.md`](docs/indextts.md)。

## 验收标准（Phase 1）

- 一条命令生成 1080×1920 MP4、SRT、封面和 JSON 报告
- 同一个 manifest 可重复执行，已有中间产物可复用
- 缺素材、缺音频、FFmpeg 失败时返回明确错误，不生成假成功结果
- 示例项目不包含真实账号、Cookie、私有 URL 或受限素材
- Windows CI 至少完成 manifest、时间线和校验层测试；FFmpeg 集成测试可按环境标记
